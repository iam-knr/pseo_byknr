About the Plugin

What is PSEO PRO by KNR?

PSEO PRO is an open-source WordPress plugin built for SEO professionals, digital marketers, and agencies who need to create large volumes of highly targeted pages from structured data — without writing a single line of code.

You bring the data (a CSV, a Google Sheet, a JSON feed, or a REST API). You build one template page. PSEO PRO does the rest — replacing {{placeholders}} with your data to generate hundreds or thousands of unique, SEO-ready pages instantly.

Each generated page gets its own title tag, meta description, canonical URL, robots tag, and JSON-LD schema markup — all derived automatically from your data row.

Update your data source and PSEO PRO auto-regenerates only the changed rows, updates existing pages, and removes orphaned pages that no longer exist in the data.

Under the Hood

How It Works

1

📥 Data Fetch

Connects to your data source — CSV URL, uploaded CSV, Google Sheet, JSON endpoint, or paginated REST API. Parses everything into a row-by-row array where each row becomes one WordPress page. No row limits, no timeouts with WP-CLI.

CSV · Google Sheets · JSON · REST API
2

🔍 Row Hashing & Smart Diff

Each row is MD5-hashed and stored in wp_pseo_data_rows. On every sync, only rows whose hash has changed are re-processed. New rows create new pages. Changed rows update existing pages. Deleted rows trigger orphan detection — the corresponding WordPress page is removed automatically.

3

🧩 Template Engine

Your WordPress page is cloned per row. Every {{placeholder}} in the title, content, URL slug, meta title, meta description, and schema fields is replaced with that row's column value. Supports {{raw:col}} for HTML output, {A|B|C} spintax for variation, and [if:col=val]...[/if] conditional blocks.

4

🚀 Page Creation & SEO Injection

Pages are created via wp_insert_post(). The SEO class hooks into wp_head at priority 1 to inject the title tag, meta description, robots directive, canonical URL, and full JSON-LD schema — all specific to that row's data. Works alongside Yoast SEO and Rank Math without conflicts.

JSON-LD Schema injected per page
5

🗺️ Sitemap & Auto-Sync

All generated pages are listed in a custom XML sitemap at /pseo-sitemap.xml ready to submit to Google Search Console. A WP Cron event (pseo_cron_sync) runs on your chosen schedule — hourly, daily, or weekly — re-fetching data and regenerating changed pages completely hands-free.

Template Engine

Syntax Reference

{{column}}

Inserts the HTML-escaped value of the CSV column named column.

{{city}} → "Bangalore"
{{raw:column}}

Inserts the raw, unescaped value — for HTML content stored inside the CSV.

{{raw:description}} → <p>Full HTML</p>
{A|B|C}

Spintax — randomly picks one option per page generation for content variation.

{Best|Top|Leading} plumber → "Top plumber"
[if:col=val]...[/if]

Conditional block — shows content only if the column matches the given value.

[if:city=Bangalore]Local offer[/if]
[if:col>0]...[/if]

Numeric comparison — supports =, !=, >, <, >=, <= operators.

[if:rating>=4]Top Rated[/if]
{{service}}/{{city}}

URL pattern field — use placeholders in the slug for clean, keyword-rich URLs.

→ /plumbing/bangalore/

Step-by-Step Guide

From Zero to 1,000 Pages

Follow these 7 steps to launch your first programmatic SEO campaign.

1

Install & Activate the Plugin

Download the zip from GitHub and upload via WP Admin → Plugins → Add New → Upload Plugin. After activation, go to Settings → Permalinks → Save Changes to flush rewrite rules — this step is mandatory or all generated pages will return 404.

💡 WP-CLI: wp plugin install pseo-pro-knr.zip --activate && wp rewrite flush
2

Prepare Your Data Source

Create your dataset with column headers in row 1 — these become your {{placeholder}} names. Choose your source type:

  • CSV via URL — host a public .csv file and paste its direct link
  • Google Sheet — share as "Anyone with link → Viewer", copy the Sheet ID from the URL (between /d/ and /edit)
  • JSON URL — point to any JSON array, specify dot-notation data path
  • REST API — configure URL, pagination param, and optional Bearer token
💡 Google Sheets export URL: docs.google.com/spreadsheets/d/SHEET_ID/export?format=csv&gid=0
3

Build Your Template Page

Create a new WordPress page using Elementor, Divi, Gutenberg, or plain HTML. Write content using {{column_name}} placeholders wherever your data should appear.

  • Use {{city}} in H1 headings, paragraphs, and image alt text
  • Use {{raw:description}} for HTML-heavy content columns
  • Use {Best|Top|Leading} spintax for natural content variation
  • Use [if:state=Karnataka]...[/if] for region-specific content blocks
💡 Tip: Set the template page status to Draft — it's the master template, not a public page.
4

Create a New PSEO Project

Go to PSEO PRO → New Project in WP Admin. Fill in all settings:

  • Project Name — internal label only (not shown publicly)
  • Generate as Post Type — Page, Post, or any custom post type
  • Content Template — select the Draft page you built in Step 3
  • Data Source — pick source type and enter URL / Sheet ID / API config
  • URL Pattern — e.g. {{service}}/{{city}} → /plumbing/bangalore/
  • SEO Title — e.g. {{service}} in {{city}} — From ₹{{price}}
  • Meta Description — 120–160 chars using placeholders
  • Schema Type — choose one of the 6 built-in JSON-LD types
  • Sync Interval — Manual / Hourly / Daily / Weekly
5

Preview Your Data First

Before generating, always click 👁 Preview Data. This fetches your source live and shows the first 5 rows, total row count, and all detected column names — so you can verify placeholder names match exactly (case-sensitive).

⚠️ Common mistake: CSV has column City (capital C) but template uses {{city}} (lowercase). Always preview first and fix mismatches before generating.
6

Generate Pages

Click ⚡ Generate Pages Now. PSEO PRO fetches all rows, applies the template, and creates or updates WordPress pages in bulk. For 10,000+ pages, use WP-CLI to bypass PHP timeout limits:

  • wp pseo list — view all projects and their numeric IDs
  • wp pseo generate --id=1 — generate project #1 from terminal
  • wp pseo generate --all --delete-orphans — all projects + clean removed rows
7

Submit Sitemap to Google Search Console

Your sitemap is automatically available at https://yourdomain.com/pseo-sitemap.xml. Go to Google Search Console → Sitemaps and submit it. Google will begin crawling and indexing all generated pages.

  • Enable Auto-Sync to keep pages updated as your data source changes
  • Use Delete Orphans to remove pages when rows are deleted from data
  • Monitor indexing in GSC under Coverage → Valid pages
💡 Pro Tip: Set Sync to Daily + Google Sheet as source. Update the sheet → PSEO PRO auto-regenerates overnight. Fully hands-free, zero manual work.
